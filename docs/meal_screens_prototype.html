Design Notes for Stakeholder Review

  • Key decision (Healthify UX): Meals tile replaces unused Steps tile in MetricsGrid. Same "+" button pattern as BP/Glucose — no new mental model needed.
  • Same sibling pattern: Tap Meals [+] → bottom sheet modal → choose Quick Select or Photo. Identical to how BP/Glucose readings work.
  • MealSummaryCard (new): Shows today's meals as colored badges between AI Insight and MetricsGrid. "Log more" link for easy access.
  • No FAB, no new tab: Healthify recommends against both. 5 bottom tabs is already the elderly cognitive max. FABs confuse Bihar grandmothers.
  • Two-tap flow: Meals [+] → Quick Select → tap category → done (3 taps including modal).
  • Hindi-first design: Hindi labels 20px bold (primary), English 14px secondary. All text via AppLocalizations.
  • 72dp touch targets: Each meal button is 72px tall, well above 48dp minimum for elderly users.
  • Color-blind safe: Every badge has a distinct icon shape (check, minus, warning, exclamation) alongside color.
  • Color consistency fix: HIGH_CARB = amber (caution) everywhere. SWEETS = red (danger) everywhere. Consistent across Quick Select and Result screens.
  • Photo path (secondary): Camera → Gemini Vision classifies → result with tip. 5-second timeout auto-falls back to Quick Select.
  • "Not correct? Change": Now OutlinedButton with visible border (Healthify requirement — must look tappable).
  • Disclaimer: Present on Quick Select and Result screens. Required for NMC compliance.
  • Colors from AppColors: primary=#0EA5E9, success=#10B981, amber=#F59E0B, danger=#EF4444, bgPage=#F0F9FF, textPrimary=#1E293B.
