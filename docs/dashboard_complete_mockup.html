Design Decisions Summary

  • Heart stays at #1 — emotional anchor for 65yo Bihar patients
  • Vitals 2x2 grid — BP, Sugar, SpO2, Steps all visible without scrolling
  • BMI kept visible — compact row below vitals (not hidden, but proportional to usage)
  • Meal slots — B / L / S / D slots prompt elderly users to log each meal
  • AI collapsed — 2 lines + expand. No more "Stage 2" clinical jargon
  • Device status card — shows Armband/Phone/BP Monitor connection state
  • 90-day trends — pushed down, now includes SpO2 + Steps trends
  • Quick actions — added "Pair Device" for easy armband setup
  • Food emoji — replaced ❗ with 🍜 (no shaming)
  • SpO2 + Steps tiles — show data source ("via Armband" / "via Phone")
  • Steps progress bar — shows % of daily goal (7,500 default for elderly)

Open Questions for Review

  • Steps goal — Default 7,500 for elderly? Should doctor set custom goal per patient?
  • SpO2 alert threshold — Show red alert below 94%? Below 90%? (Clinical standard: <94% is concern)
  • No armband state — Show "Connect Armband" prompt in SpO2/Steps tiles? Or hide tiles entirely?
  • Meal slot count — 4 slots (B/L/S/D) or 3 slots (B/L/D) with optional snack?
  • Steps from phone — Use Android Health Connect? Works on budget Redmi/Realme?
